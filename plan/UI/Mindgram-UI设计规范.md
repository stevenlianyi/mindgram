# Mindgram UI 设计规范

> 版本: v1.2 | 日期: 2026-06-09 | 平台: 微信小程序

---

## 一、设计理念

### 1.1 核心设计原则

| 原则 | 说明 |
|------|------|
| **情感优先** | 色彩、动效、微交互都以传递情绪温度为目标，而非冷冰冰的工具感 |
| **轻量社交** | Emoji 作为视觉主角，降低表达门槛；界面不打扰用户真实感受 |
| **渐进呈现** | 从日常打卡的轻量交互，逐步深入到周/季/年的厚重叙事 |
| **治愈感** | 柔和的圆角、毛玻璃、渐变背景，传递安全与舒适 |

### 1.2 品牌关键词

温暖 · 趣味 · 连接 · 自我认知 · 成长

## 六、交互规范

### 6.1 导航交互

| 交互 | 触发方式 | 动画 |
|------|---------|------|
| Tab 切换 | 点击底部 Tab | 图标缩放+颜色渐变 200ms |
| 页面跳转 | navigateTo | 右滑推入 300ms |
| 返回 | navigateBack / 物理返回 | 右滑退出 300ms |
| 下拉刷新 | 手势下拉 | 弹性回弹 + Loading |
| 弹窗出现 | 点击触发 | 底部滑入 / 缩放淡入 250ms |
| 弹窗关闭 | 点击遮罩/关闭 | 反向动画 200ms |

### 6.2 反馈交互

| 场景 | 反馈方式 |
|------|---------|
| **打卡成功** | Haotic 撒花动画 + 连续天数+1 火焰效果 + 轻振动 |
| **猜谜正确** | 金色粒子 +10积分飘字动画 + 轻振动 |
| **猜谜错误** | 正确答案揭晓 + 鼓励文字 + 轻微抖动 |
| **获得徽章** | 全屏徽章揭幕动画 + 庆祝粒子 |
| **加载中** | 骨架屏 (优先于 Spinner) |
| **操作失败** | Toast 红色提示 2秒 |
| **操作成功** | Toast 绿色提示 1.5秒 |
| **空状态** | 插画 + 友好引导文案 |

### 6.3 手势交互

| 手势 | 场景 |
|------|------|
| 下拉 | 刷新动态流/列表 |
| 上拉 | 加载更多 |
| 双指缩放 | 自拍贴图缩放 Emoji |
| 单指拖拽 | 移动 Emoji 贴图位置 |
| 长按 | 快捷操作菜单 |
| 左右滑动 | Tab/分段切换，图片浏览 |

## 七、动效规范

### 7.1 动效时长

| 类型 | 时长 | 缓动 |
|------|------|------|
| 微交互 (按钮点击) | 150ms | ease-out |
| 页面过渡 | 300ms | ease-in-out |
| 弹窗/浮层 | 250ms | ease-out |
| 列表项出现 | 300ms 依序延迟 50ms | ease-out |
| 粒子/庆祝动画 | 1500ms | ease-out |
| 加载动画 | 循环 | linear |

### 7.2 关键动效列表

- **Emoji 选中**: scale(1.0)→scale(1.15) 弹跳 + 周围光晕
- **打卡火焰**: 渐变填充 + 火苗摆动循环动画
- **积分飘字**: +10 从中心上飘+淡出
- **徽章揭幕**: 全屏暗色→光束→徽章旋转展示
- **猜谜正确**: 选项卡片绿色高亮→粒子爆炸
- **列表加载**: 骨架屏 wave 动画→内容 fadeIn
- **图片加载**: 模糊预览→清晰渐显 (lazy load)
- **Tab 切换**: 指示器平滑滑动 250ms
